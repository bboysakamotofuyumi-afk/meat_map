
# Project: Tokyo Carnivore Meat Map

## Goal
東京都内の「カーニボア向けの肉料理が食べられる店」を、外部APIから自動収集し、
カーニボア度をスコアリングして Google マイマップにインポート可能な CSV を生成する。

## Scope
- 対象エリア: 東京都
- データソース: ホットペッパーグルメAPI（Google Places API はコストの都合で現在停止中／必要なら再開を検討）
- 出力: マイマップ用 CSV（店名, 住所, lat/lng, カーニボアランク, ジャンル, rating 等）

## What I want you (Codex) to do
1. 東京都をグリッド分割するユーティリティや今後の拡張に備えた構造を維持しつつ、現在はホットペッパーのみから肉系飲食店を取得するコードを書く。
2. 複数ソースの店舗データを統合し、共通スキーマにマッピングする（将来的に別ソースが追加されても対応できる前提）。
3. カーニボアスコアをルールベースで計算し、S/A/B ランクを付与する。
4. マイマップに読み込める CSV を出力する CLI を作る。
5. 上記をテストしやすい構造（モジュール分割＋簡単なテスト）で実装する。

## Constraints / Conventions
- 言語: Python
- 依存関係は `requirements.txt` に追加する。
- APIキーはコードに直書きせず、環境変数から読み込む。
- 実装ファイルは `meatmap/` 以下に配置する。
